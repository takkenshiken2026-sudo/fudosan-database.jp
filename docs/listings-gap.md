# 募集価格 × 成約価格ギャップ

現在の**募集（売出し）価格**を独自に蓄積し、国土交通省 不動産情報ライブラリの
**成約価格**と突き合わせて「乖離率」「想定値引き率」を算出する機能です。

無料APIに存在しない「今いくらで売り出されているか」を持つことが、このサイトの
競争優位になります。買い手には指値の目安、売り手には売り出し価格の妥当性、
仲介には市況の裏付け資料になります。

## データの入手（重要・合法性）

募集価格は **CSV インポート方式** で受け入れます。各不動産ポータルからの無断
スクレイピングは規約違反のため行いません。想定する合法な入手経路:

- 不動産会社・データ事業者との**提携フィード / API 契約**
- 自治体などの**オープンデータ**
- 自社サイトに掲載してもらった**一次データ**
- 許諾を得た範囲での**手動登録**

`source` 列に取得元を必ず記録し、出所を追跡できるようにしています。

## CSV フォーマット

`backend/app/listings/sample_listings.csv` を参照。

| 列 | 必須 | 説明 |
|----|------|------|
| `source` | ✓ | 取得元の識別子（例: `partner_a`, `opendata`, `manual`） |
| `external_id` | ✓ | 取得元での物件ID（重複判定に使用） |
| `municipality_code` | ✓ | 市区町村コード（5桁、例: `13113`） |
| `observed_date` | ✓ | 観測日（`YYYY-MM-DD`）。同一物件を定点観測する想定 |
| `listing_price` | ✓ | 募集価格（円） |
| `property_type` |  | 物件種別。成約データと同じ表記推奨（例: `中古マンション等`, `宅地(土地)`） |
| `district_name` |  | 地区名 |
| `area` |  | 面積（㎡）。あれば㎡単価を自動計算 |
| `unit_price` |  | ㎡単価（円）。未指定かつ area があれば自動算出 |
| `building_year` |  | 築年（例: `2015年`） |
| `floor_plan` |  | 間取り（例: `2LDK`） |
| `first_listed_date` |  | 初回掲載日（滞留日数の将来集計用） |
| `is_active` |  | 掲載中フラグ（既定 `true`）。集計は掲載中のみ対象 |

一意キーは `(source, external_id, observed_date)`。同じ物件を別日に観測した
レコードは別行として蓄積され、将来の値下げ履歴・滞留日数の分析に使えます。

## 取込・集計コマンド

```bash
cd backend

# CSV取込＋集計再計算（取込後に自動でrebuild）
python -m app.sync_cli import-listings --csv app/listings/sample_listings.csv

# 集計のみ再計算（特定市区町村）
python -m app.sync_cli rebuild-listing-stats --city 13113

# 件数確認
python -m app.sync_cli status   # listing_snapshots / listing_stats が増える
```

## 表示

- ページ: `/gap/{都道府県slug}/{市区町村slug}`（例: `/gap/tokyo/shibuya-ku`）
- 募集データが無い市区町村は成約相場ページ（`/price/...`）へリダイレクト
- 募集データがある市区町村は、相場ページに「募集×成約ギャップ」ボタンが表示され、
  サイトマップにも `/gap/...` が追加されます

## 算出ロジック

- 物件種別ごとに、**最新の観測四半期**の募集平均と、**直近3年**の成約平均（`price_classification='01'`）を比較
- ㎡単価が両側にあれば単価ベース、なければ総額ベースで算出
  - 乖離率 = (募集 − 成約) / 成約 × 100
  - 想定値引き率 = (募集 − 成約) / 募集 × 100
- 募集・成約それぞれ 3 件以上あるときだけ表示（サンプル過少を除外）
