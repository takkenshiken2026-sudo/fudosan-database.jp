# SEO 戦略 — 平均掲載順位の改善方針

> GSC「Performance on Search」(〜2026-07) の分析に基づく方針と、実装済みの施策メモ。

## 結論：追うべき KPI

DB 型（プログラマティック）サイトでは、ページが増えインデックスされるほど
40 位台のロングテール表示が積み上がり、**サイト全体の平均掲載順位は構造的に下がる**。
これは弱さではなく露出が広がっている証拠であり、SUUMO / HOME'S 等の大手でも
サイト全体平均は 20〜40 位台になる。

したがって「サイト全体の平均順位 ≤ 5 位」は追わない（追うと大量ページの
noindex＝露出削減が最短手段になり本末転倒）。代わりに以下を追う。

| ❌ 追わない | ✅ 追う |
|---|---|
| サイト全体の平均掲載順位 | **top5 / top10 に入るクエリ数** |
| 全表示回数の加重平均 | **クリック数・CTR** |
| — | 主要クエリ群だけに絞った平均（GSC でページ/クエリを絞って計測） |

## データ診断（初期 2 週間）

- データはまだ約 2 週間分。判断は時期尚早。
- 平均順位が 7→37 位に「悪化」して見えるのは、Google が新規ページを
  次々発見し多数のクエリでテスト配信し始めた正常な評価期。
- 総クリック 1・総表示 158＝CTR ほぼ 0。まだ 1 ページ目に入れていないのが主因。
- クエリは全て「〇〇 相場 / 家賃相場 / 土地価格 / 不動産価格」の地域×意図型。狙いは正しい。

### 検出したボトルネック

1. **「家賃相場」クエリで上位表示されるが家賃データが無い**（意図とコンテンツの不一致）→ 当面は狙わない。
2. **タイトル/H1 とクエリの語ズレ**（「土地価格」「相場」が title に不足）→ 実装で解消。
3. **FAQ 構造化データが無い**（リッチリザルト/CTR 機会損失）→ 実装で追加。
4. **内部リンクが同一都道府県内のみ**（隣接エリア/同意図ページへの横リンクが弱い）→ 今後の課題。
5. **本文が数値・グラフ中心で、検索意図に答える地の文が薄い** → 冒頭要約を実装。

技術基盤（canonical / breadcrumb / schema.org Dataset / sitemap / robots）は良好。

## 実装済みの施策（このブランチ）

`backend/app/web/`:

- **content.py（新規）**: DB 数値から、市区町村・都道府県ページの
  冒頭要約テキストと FAQ（可視セクション兼 FAQPage 構造化データ）を動的生成。
- **seo.py**: title / meta description をクエリ語（相場・土地価格・地価）＋
  動的な数値（平均取引価格・地価・前年比・年）で強化。`FAQPage` を構造化データに追加。
- **templates/municipality.html・prefecture.html**: H1 に「相場・土地価格」を反映、
  冒頭要約テキスト、可視 FAQ セクションを追加（FAQPage schema と語一致）。

いずれもデータが無い場合はスキップし、事実に基づく文言のみを出力（新規・データ薄の
ページでも安全）。本番の静的サイトは `TestClient(app)` 経由でこれらのルートを
描画するため、変更はそのまま反映される。

## 今後のロードマップ

| 時期 | やること | 期待 |
|---|---|---|
| 〜1ヶ月 | 実装反映・sitemap 再送信・GSC でインデックス精査 | CTR 改善の土台 |
| 1〜3ヶ月 | 勝てる主要 100 クエリへ内部リンク集中、隣接エリア相互リンク、コンテンツ拡充 | 主要クエリ 40 位台→10〜20 位 |
| 3〜6ヶ月 | 被リンク獲得（データ引用促進）、継続更新（lastmod）、勝ち筋の横展開 | 主要クエリで **top5** 出現 |

**「主要クエリで top5」は 3〜6ヶ月で射程内。** ただしサイト全体平均は
20 位前後で頭打ちになるのが健全、という前提で運用する。
